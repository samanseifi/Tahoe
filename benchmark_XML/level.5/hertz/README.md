# Hertz contact benchmark — explicit + implicit Tahoe

Quarter-symmetry hemisphere pressed onto a stiff base.  Designed to
exercise both Tahoe contact pathways on the same mesh:

- **explicit** — `<explicit_solid>` + central-difference + lumped mass +
  `<contact_3D_penalty>` with normal viscous damping (#31), driving
  toward the quasistatic Hertz limit through a smooth ramp.
- **implicit** — classic `<updated_lagrangian>` + Newton–Raphson +
  `<contact_3D_penalty>` (no damping needed; Newton is quadratically
  convergent, ~3–4 iterations per load step).

Both produce a contact-pressure profile p(r) that follows the Hertz
analytical shape; quantitative agreement on contact radius `a`, total
load `F`, and peak pressure `p₀` is in the 5–25 % range — typical for
penalty contact at this mesh density.

## Geometry

Generated by [generate_hertz_mesh.py](generate_hertz_mesh.py).  Inline
Tahoe `.geom` with two element blocks (block 1 = sphere, block 2 = base)
and 8 node sets / 2 side sets:

```
R         = 10.0 mm   sphere radius
R_box     = 0.5*R     quarter-square edge length (clear of sphere edge)
H_base    = 5.0 mm    base block thickness
gap       = 1e-3 mm   initial gap (avoids zero-penetration edge case)
nx=ny=24, nz_sph=12, mx=my=24, mz_base=8
total: 11 520 hex8, 13 750 nodes
```

tanh grading concentrates resolution near the contact apex (smallest
in-plane element ≈ 0.12 mm; the analytical Hertz contact radius at
δ = 0.05 mm is a ≈ 0.71 mm — about 6 elements across).

Node sets:
| ID | description                                     |
|----|--------------------------------------------------|
| 1  | sphere top (driven by prescribed δ)             |
| 2  | sphere x = 0 (symmetry, u_x = 0)                |
| 3  | sphere y = 0 (symmetry, u_y = 0)                |
| 4  | base bottom (clamped)                           |
| 5  | base x = 0  (symmetry)                          |
| 6  | base y = 0  (symmetry)                          |
| 7  | sphere bottom (contact strikers)                |
| 8  | base top    (contact face nodes)                |

Side sets:
| ID | block | face | description                |
|----|-------|------|----------------------------|
| 1  | 1     | 1    | sphere bottom (-z)         |
| 2  | 2     | 2    | base top (+z)              |

## How to run

```bash
cd benchmark_XML/level.5/hertz
python3 generate_hertz_mesh.py             # writes geometry/hertz.geom
../../../build/bin/tahoe -f hertz_explicit.xml    # ~24 min, 12 threads
../../../build/bin/tahoe -f hertz_implicit.xml    # ~5 min,  serial
python3 compare_to_analytical.py            # writes hertz_pressure.png
```

## Material parameters

Both XMLs use `Simo_isotropic` with E = 200 000, ν = 0.3, ρ = 7.85 (in
the consistent unit system Tahoe expects: E in MPa-numerical,
ρ in mass-numerical such that c_p = √(E_eff/ρ) is finite-O(100)).

The **explicit** benchmark uses **identical material** in both blocks
(E_base = E_sphere) so a single `<large_strain_element_block>` covers
both.  Effective contact modulus:

  E* = E / (2(1 − ν²)) ≈ 109 890

The **implicit** benchmark uses a stiffer base (E_base = 10 × E_sphere),
which approximates a near-rigid half-space and gives:

  1/E* = (1−ν²)/E_sphere + (1−ν²)/E_base
  E*  ≈ 199 800

This matters: the contact-pressure peak scales linearly with E*, and
the matched Hertz curves below differ between the two runs accordingly.

## Boundary conditions

Sphere top (NS1): u_z = δ_BC × schedule(t),  with δ_BC = −0.05 mm.

Symmetry: NS2/NS3 (sphere) and NS5/NS6 (base) fix the normal in-plane
displacement to zero.  Base bottom (NS4) is fully clamped.

## Driving schedule

- **explicit**: smooth half-cosine ramp 0 → 1 over t ∈ [0, 1] with 11
  control points; T = 1.0 contains ≈ 16 wave traversals (R/c_p ≈ 0.063);
  Δt = 1×10⁻⁴ (0.5× CFL); 10 000 steps.  Viscous damping
  `viscous_damping = 2.0` on the contact pair suppresses elastic
  ringing so the late-time state approximates quasistatic.
- **implicit**: 10 monotonic load steps, Newton–Raphson with
  rel_tolerance = 1e-8, ~3–4 iterations/step, quadratic convergence.

## Results (δ_BC = 0.05 mm)

```
  run              E*    a_sim     a_Hz   F_quarter      F_Hz/4   p₀_sim    p₀_Hz
  explicit     109890   0.829    0.707      955  N      1295 N    9384      4947
  implicit     199800   0.733    0.707     2161  N      2355 N   14414      8994
```

- **Contact radius** matches √(R·δ_BC) within 4 % (implicit) – 17 %
  (explicit).  The slight overshoot is expected: the penalty admits
  small finite penetration outside the Hertz analytical zone.
- **Total load F_quarter** is within 8 % of analytical for the implicit
  run; the explicit run is ~75 % of analytical because the soft
  penalty (`fK = 1e5`) and viscous damping leave a small residual
  oscillation that biases the load downward in the late-time average.
- **Peak nodal pressure p₀_sim** overshoots analytical by ~2× — a
  well-known artifact of point-collocation penalty contact (the apex
  striker concentrates load into a small tributary area).  The
  *integrated* load and the *off-apex* pressure profile match Hertz
  much more closely.
- **δ_apex_sim** (sphere apex DZ − base apex DZ) is small (≈ 10 μm
  explicit, ≈ 1.5 μm implicit) because **both** bodies deform together
  under the prescribed top displacement.  The relevant Hertz parameter
  here is δ_BC (the imposed indenter motion), since the curved sphere
  pushed down by δ_BC produces a contact zone of size √(R·δ_BC)
  regardless of how the bulk distributes elastic strain.

## Limitations

- The base block has clamped bottom at z = −5 mm — this is a
  **finite-thickness** elastic block, not a true half-space.  The
  bulk-distributed compression of base + sphere absorbs part of the
  prescribed top displacement, so δ_apex (apex-to-apex relative
  motion) is much smaller than δ_BC.
- The explicit run is dynamic: the 16-traversal damping window is
  enough to bring per-step KE below 1 % of contact PE, but not zero.
  The implicit run is the cleaner quasistatic answer.
- Penalty contact gives ~1-element-thick smearing of the pressure
  drop at r = a.  Increasing `penalty_stiffness` tightens the zone
  but raises peak nodal pressure; decreasing it does the opposite.
  For an essentially mesh-converged Hertz match, both `penalty_stiffness`
  and the tangential resolution `nx,ny` would need to be increased
  together (and the explicit Δt scaled with √fK).
